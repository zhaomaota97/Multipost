# 打包成 Mac 程序并分发给朋友

本工具用浏览器自动化操作平台，**必须在某台电脑本地运行**。给 Mac 朋友用的最稳方式：
打包成 `.app`，他双击本地运行（看不到源码、无需装 Python、无需服务器、无风控/信任问题）。

由于 PyInstaller 不能跨平台编译（Windows 上无法产出 Mac 程序），这里用 **GitHub 免费的
macOS 云端构建**来打包。

## 一、构建（一次性设置）

1. 在 https://github.com 注册/登录，新建一个**私有仓库**（Private），名字随意，比如 `fabu`。
2. 把本项目代码推上去（见下方"推送命令"）。
3. 打开仓库页面 → **Actions** 标签 → 左侧选 **build-mac** → 右上 **Run workflow** → 绿色按钮运行。
4. 等约 5–10 分钟，构建完成后在该次运行页面底部 **Artifacts** 下载：
   - `视频一键发布-apple-silicon`（M 系列芯片 Mac，2020 年后多为此）
   - `视频一键发布-intel`（Intel 芯片 Mac）
   按朋友的机型给对应的那个。不确定就让他在「关于本机」看芯片。

## 二、朋友首次使用（重要）

`.app` 没有花钱做苹果签名，macOS 默认会拦截。让朋友这样打开：

1. 解压下载的 zip，得到 `视频一键发布.app`，拖到「应用程序」或桌面。
2. **右键点击该 .app → 选「打开」→ 在弹窗里再点「打开」**（只需首次这样，之后可直接双击）。
   - 若提示"已损坏/无法验证开发者"，在「系统设置 → 隐私与安全性」页面点「仍要打开」；
     或在终端执行：`xattr -cr "/Applications/视频一键发布.app"`。
3. 启动后会自动打开浏览器到 `http://127.0.0.1:8800`，照常使用。
4. 他的登录态、日志、视频等数据保存在用户主目录下的 **`视频一键发布数据`** 文件夹里。
5. 退出：在程序坞图标上右键 → 退出。

## 三、推送命令（把代码传到 GitHub）

在本项目目录执行（把 URL 换成你新建仓库的地址）：

```bash
git add -A
git commit -m "video one-click publisher"
git branch -M main
git remote add origin https://github.com/你的用户名/fabu.git
git push -u origin main
```

推送需要 GitHub 登录凭据（用户名 + Personal Access Token，作为密码粘贴）。
Token 在 GitHub → Settings → Developer settings → Personal access tokens 生成（勾选 repo 权限）。

## 四、以后更新

改完代码后再次 `git push`，到 Actions 重新 Run workflow，下载新产物给朋友即可。

> 说明：朋友机型不同需要不同芯片的包；x86_64(intel) 包在 Apple Silicon 上也能借 Rosetta 运行，
> 但原生 arm 包更快更省心。
